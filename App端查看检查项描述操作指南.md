# App端查看检查项描述操作指南

## 📱 完整导航路径

### 方法一：通过首页进入（推荐）

```
首页 → 我的工单 → 选择工单 → 查看检查详情 → 继续检查 → 点击任意检查项
```

**详细步骤：**

1. **登录App**，进入首页

2. **点击"我的工单"**（或统计卡片中的工单数量）
   - 位置：首页中间的统计卡片区域
   - 图标：🔍 或 📋

3. **选择一个工单**
   - 在工单列表中点击任意待处理或进行中的工单

4. **查看检查详情**
   - 工单详情页会显示关联的检查记录
   - 点击进入检查详情

5. **点击"继续检查"按钮**
   - 位置：检查详情页底部
   - 按钮文字：**"继续检查"**（如果是被驳回的状态会显示"修改检查结果"）

6. **进入检查项列表页面**
   - 这里会显示所有检查项
   - 每个检查项都有名称、状态、扇区信息等

7. **点击任意检查项**
   - 点击列表中任何一个检查项

8. **查看检查项描述** ✨
   - 在弹出的详情窗口中
   - 在"基本信息"部分的下方
   - 会看到一个**黄色背景的区域**，显示：
     ```
     💡 检查说明
     [这里是检查项的描述信息]
     ```

---

### 方法二：通过检查列表进入

```
首页 → 现场检查 → 选择检查记录 → 继续检查 → 点击任意检查项
```

**详细步骤：**

1. **首页** → 点击快捷操作区的 **"现场检查"** 📷

2. **检查列表页面**
   - 查看所有检查记录
   - 点击任意一个检查记录

3. **检查详情页**
   - 点击底部的 **"继续检查"** 按钮

4. **检查项列表**
   - 点击任意检查项查看详情

5. **查看描述信息** ✨

---

## 🎯 识别描述信息的特征

当您点击检查项后，在弹出的详情窗口中，描述信息有以下特征：

### 视觉特征：
- **位置**：在"基本信息"卡片的底部
- **图标**：💡 灯泡图标
- **标题**："检查说明"（橙色文字）
- **背景**：淡黄色渐变背景
- **边框**：左侧有橙色竖线边框

### 示例外观：
```
┌─────────────────────────────────────┐
│ 💡 检查说明                          │
│                                     │
│ 使用GPS确认站点实际坐标。需要记录   │
│ 经纬度信息，精度要求±10米以内。     │
│ 测量时请确保GPS信号良好。           │
└─────────────────────────────────────┘
```

## 📝 什么时候能看到描述？

### ✅ 能看到描述的情况：
1. **检查项有配置描述**：在WebAdmin中模板配置时已添加描述
2. **新创建的检查**：使用新模板创建的检查
3. **已更新的检查**：运行过更新脚本的现有检查

### ❌ 看不到描述的情况：
1. **检查项没有配置描述**：模板中该检查项的描述为空
2. **非常旧的检查**：在功能开发之前创建且未运行更新脚本

## ❓ 常见问题

### Q1: 我点击了检查项，但看不到黄色的描述区域？

**可能原因：**
1. 该检查项在模板中没有配置描述
2. 需要向下滚动才能看到（描述在基本信息下方）
3. 检查是在更新之前创建的

**解决方法：**
1. 检查WebAdmin中该检查项是否有描述配置
2. 在检查项详情窗口中向下滚动
3. 创建一个新的检查记录测试

### Q2: 部分检查项有描述，部分没有？

**这是正常的**：
- 只有在WebAdmin模板编辑器中配置了描述的检查项才会显示
- 可以去WebAdmin补充缺失的描述

### Q3: 描述信息可以修改吗？

**描述信息来源于模板**：
- 需要在WebAdmin的"检查模板管理"中修改
- 修改后，新创建的检查会使用新描述
- 已创建的检查不会自动更新（可以运行更新脚本）

### Q4: 我想批量更新现有检查项的描述？

**运行更新脚本**：
```bash
cd backend
python update_existing_check_items_description.py
```

## 📊 验证是否配置成功

### 在WebAdmin中验证：
1. 进入"检查模板管理"
2. 编辑任意模板
3. 展开检查分类和检查项
4. 查看"检查项描述"输入框是否有内容

### 在App端验证：
1. 创建一个新的检查
2. 进入检查项列表
3. 点击任意检查项
4. 查看是否显示黄色描述区域

### 在数据库中验证：
```bash
cd backend
sqlite3 site_manager.db "SELECT item_name, description FROM inspection_check_items WHERE description IS NOT NULL AND description != '' LIMIT 5;"
```

## 💡 最佳实践建议

### 对于管理员：
1. **完善模板描述**：为所有检查项添加清晰的描述
2. **包含关键信息**：
   - 填写要求
   - 取值范围
   - 注意事项
   - 操作指引
3. **保持一致性**：使用统一的描述格式

### 对于检查员：
1. **养成查看描述的习惯**：填写前先看描述说明
2. **反馈问题**：如果描述不清楚，及时反馈给管理员
3. **按描述要求操作**：严格按照描述中的要求进行检查和填写

## 📞 需要帮助？

如果按照上述步骤仍然看不到描述信息，请检查：

1. ✅ 后端服务是否重启（如果刚修改代码）
2. ✅ 数据库是否运行了迁移脚本
3. ✅ 模板中是否配置了描述
4. ✅ 检查记录是否是新创建的或已更新

如果问题仍未解决，可以查看：
- 后端日志
- 浏览器控制台（如果是H5版本）
- 数据库中的实际数据
