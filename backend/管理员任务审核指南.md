# 管理员任务审核指南

## 概述
安装人员提交任务后，管理员可以通过以下流程进行审核和检查。

## 审核流程

### 1. 查看待审核任务

管理员可以通过以下方式查看待审核任务：

**方法一：通过任务列表筛选**
- 登录管理员账号
- 进入任务列表页面
- 筛选状态为"已提交"(submitted)的任务

**方法二：使用API查询**
```bash
GET /api/tasks/?status=submitted
```

### 2. 查看任务详情

点击任务卡片进入任务详情页面，管理员可以看到：

**任务基本信息：**
- 任务标题和类型
- 关联站点信息
- 执行人员信息
- 任务优先级和时间信息

**任务进度：**
- 进度条显示完成百分比
- 进度步骤时间线

**检查记录：**
- 关联的检查记录列表
- 每条检查记录的状态和完成率
- 点击查看详细检查项目

**现场照片：**
- 任务执行过程中拍摄的现场照片
- 照片描述和GPS信息

**操作历史：**
- 任务状态变更历史
- 操作人员和时间记录

### 3. 进行任务审核

**开始审核：**
- 在任务详情页面，点击"审核任务"按钮
- 系统会跳转到专门的任务审核页面

**审核页面内容：**
1. **任务信息确认**
   - 核对任务基本信息
   - 查看提交时间和执行情况

2. **提交内容检查**
   - 检查工作说明和备注
   - 查看检查记录详情
   - 浏览现场照片
   - 检查GPS定位信息

3. **审核决策**
   - **通过(approved)**: 任务执行合格，予以通过
   - **驳回(rejected)**: 任务执行不合格，需要重新执行

### 4. 审核操作

**审核通过流程：**
1. 选择"审核通过"
2. 填写审核意见（如：检查记录完整，任务执行良好，审核通过）
3. 点击"提交审核"
4. 任务状态变更为"已批准"(approved)

**审核驳回流程：**
1. 选择"审核驳回" 
2. 详细说明驳回原因（如：检查记录不够详细，需要重新检查设备安装情况）
3. 选择是否需要重新检查
   - 开启：任务重新分配给执行人员，状态变为"已分配"(assigned)
   - 关闭：任务状态变为"已驳回"(rejected)
4. 点击"提交审核"

### 5. 审核后处理

**审核通过后：**
- 任务进入"已批准"状态
- 可以继续完成后续流程（如标记为完成）

**审核驳回后：**
- 如果需要重新检查：任务重新分配给执行人员
- 执行人员需要重新开始检查并提交
- 管理员可以继续进行后续审核

## API 接口说明

### 审核任务 API
```http
POST /api/tasks/{task_id}/review
Content-Type: application/json
Authorization: Bearer {admin_token}

{
  "result": "approved",  // approved 或 rejected
  "comments": "审核意见",
  "require_recheck": false  // 是否需要重新检查（仅在驳回时有效）
}
```

**响应示例：**
```json
{
  "message": "任务审核approved成功",
  "task": {
    "id": "task_id",
    "status": "approved",
    "task_title": "任务标题",
    ...
  }
}
```

### 查询待审核任务
```http
GET /api/tasks/?status=submitted
Authorization: Bearer {admin_token}
```

## 权限说明

- 只有 **管理员(admin)** 和 **经理(manager)** 角色可以审核任务
- 只能审核状态为"已提交"(submitted)的任务
- 审核操作会自动记录在任务历史中

## 最佳实践

1. **仔细检查提交内容**
   - 核对检查记录是否完整
   - 查看现场照片是否清晰、数量充足
   - 确认GPS定位信息准确

2. **提供明确的审核意见**
   - 通过时：确认任务完成质量
   - 驳回时：详细说明需要改进的地方

3. **合理使用重新检查选项**
   - 轻微问题：驳回但不需要重新检查
   - 严重问题：驳回并要求重新检查

4. **及时处理待审核任务**
   - 定期查看待审核任务列表
   - 避免任务积压影响工作效率

## 状态流程图

```
[submitted] → [审核] → [approved] ✅
             ↓
           [rejected] → [需要重新检查？] → [assigned] (重新开始)
                      ↓
                    [rejected] (结束)
```
