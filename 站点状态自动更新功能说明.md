# 站点状态自动更新功能说明

## 功能概述

系统现已实现站点状态的自动更新功能，根据工单的创建和完成情况自动管理站点的生命周期状态。

## 站点状态定义

站点有以下4个状态：

| 状态 | 值 | 说明 | 显示颜色 |
|------|------|------|----------|
| 规划中 | `planning` | 站点初始状态，正在规划设计 | 灰色 |
| 建设中 | `construction` | 站点正在施工建设，设备安装调试阶段 | 黄色 |
| 运营中 | `operational` | 站点已完工并投入使用，正常运行 | 绿色 |
| 维护中 | `maintenance` | 站点需要维护或正在维护 | 红色 |

## 自动更新规则

### 规则1：创建安装工单时

**触发条件：**
- 创建工单类型为 `opening_inspection`（安装工单）
- 站点当前状态为 `planning`

**操作：**
- 自动将站点状态从 `planning` 更新为 `construction`
- 记录审计日志：`创建安装工单，站点进入建设阶段`

**示例：**
```
站点A (状态: planning) 
→ 创建安装工单 
→ 站点A (状态: construction)
```

### 规则2：完成安装工单时

**触发条件：**
- 完成的工单类型为 `opening_inspection`（安装工单）
- 该站点下所有安装工单都已完成
- 站点当前状态不是 `operational`

**操作：**
- 自动将站点状态更新为 `operational`
- 记录审计日志：`所有N个安装工单已完成，站点投入运营`

**示例场景1：单个安装工单**
```
站点A (状态: construction, 安装工单1: PENDING) 
→ 完成安装工单1 
→ 站点A (状态: operational)
```

**示例场景2：多个安装工单**
```
站点B (状态: construction, 安装工单1: PENDING, 安装工单2: PENDING)
→ 完成安装工单1
→ 站点B (状态: construction) // 仍有未完成的工单
→ 完成安装工单2
→ 站点B (状态: operational) // 所有工单完成
```

## 工单完成的触发点

站点状态更新会在以下三个API调用时触发：

1. **`POST /api/work-orders/{id}/complete`** - 手动完成工单
2. **`POST /api/work-orders/{id}/review`** - 审核通过工单（action=approve）
3. **`PUT /api/work-orders/{id}/mark-completed`** - 标记工单为完成

## 审计日志

每次站点状态自动更新都会生成审计日志，记录在 `audit_events` 表中：

| 字段 | 说明 |
|------|------|
| resource_type | 固定值 "site" |
| resource_id | 站点ID |
| action | 固定值 "status_change" |
| operator_id | 系统管理员ID（系统自动操作） |
| from_status | 变更前的状态 |
| to_status | 变更后的状态 |
| comments | 变更原因说明 |
| details | JSON格式的详细信息 |

## 实现细节

### 核心函数

**1. `_update_site_status_on_work_order_create()`**
- 位置：`backend/app/api/work_orders.py`
- 功能：工单创建时检查并更新站点状态
- 调用时机：创建工单API中，数据库提交前

**2. `_update_site_status_on_work_order_complete()`**
- 位置：`backend/app/api/work_orders.py`
- 功能：工单完成时检查并更新站点状态
- 调用时机：工单完成的所有API中，数据库提交前

**3. `_audit_site_status_change()`**
- 位置：`backend/app/api/work_orders.py`
- 功能：记录站点状态变更的审计日志
- 调用时机：状态更新时自动调用

### 关键逻辑

```python
# 判断所有安装工单是否都已完成
opening_work_orders = db.query(WorkOrder).filter(
    WorkOrder.site_id == site_id,
    WorkOrder.type == WorkOrderTypeEnum.OPENING_INSPECTION
).all()

all_completed = all(
    wo.status == WorkOrderStatusEnum.COMPLETED 
    for wo in opening_work_orders
)

if all_completed and opening_work_orders:
    # 更新站点状态为operational
    site.status = "operational"
```

## 注意事项

1. **只针对安装工单**：只有 `opening_inspection` 类型的工单会触发站点状态自动更新
2. **状态前置条件**：
   - planning → construction：只在站点为planning时才会更新
   - construction → operational：只在所有安装工单完成时才会更新
3. **维护工单不影响状态**：其他类型的工单（维护、故障等）不会触发站点状态变更
4. **手动修改优先**：管理员仍可通过API手动修改站点状态，不受自动更新限制
