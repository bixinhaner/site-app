# 检查项描述功能说明

## 功能概述

现在检查项支持显示描述信息，帮助检查人员更好地理解如何填写每个检查项。

## 使用方法

### 1. 在WebAdmin配置模板时添加描述

在`检查模板管理`页面编辑模板时，每个检查项都有一个"检查项描述"输入框：

```
检查项名称：天线安装检查
检查项描述：检查天线安装位置、角度和固定情况。需要测量方位角(0-360度)、下倾角(0-20度)和天线挂高(0-100米)
```

### 2. App端查看描述

在移动端检查列表中点击任意检查项，在详情弹窗中会显示：

```
💡 检查说明
检查天线安装位置、角度和固定情况。需要测量方位角(0-360度)、下倾角(0-20度)和天线挂高(0-100米)
```

描述区域特点：
- 💡 灯泡图标，醒目易识别
- 黄色渐变背景，与其他信息区分
- 橙色左边框，突出显示
- 支持多行文本，自动换行

## 建议的描述内容

在模板配置时，描述中可以包含：

1. **填写要求**：说明需要填写什么数据
2. **取值范围**：如"方位角：0-360度"
3. **填写示例**：如"例如：方位角 120°，下倾角 3°"
4. **注意事项**：如"注意：需要使用专业测量工具"
5. **操作指引**：如"请先拍摄天线铭牌，然后测量参数"

## 示例描述

### 好的描述示例：

```
天线方位角检查
描述：使用指南针或测量仪器测量天线的水平方向角度。
- 取值范围：0-360度
- 精度要求：±1度
- 示例：正北方向为0度，正东为90度
- 注意：测量时需站在天线正后方
```

### 不好的描述示例：

```
天线方位角检查
描述：检查方位角
```
（太简略，没有提供有用信息）

## 技术细节

### 数据库字段
- 表：`inspection_check_items`
- 字段：`description` (TEXT类型)

### 数据流程
1. WebAdmin配置模板时，`description`存储在`template_data` JSON中
2. 创建检查时，从模板提取`description`并存储到检查项
3. App端获取检查项时，自动包含`description`字段
4. 前端UI在检查项详情中显示描述

## 验证方法

### 1. 检查数据库
```bash
cd backend
sqlite3 site_manager.db "SELECT item_name, description FROM inspection_check_items WHERE description != '' LIMIT 5;"
```

### 2. 在App端验证
1. 打开任意检查记录
2. 点击任意检查项
3. 在"基本信息"下方应该能看到黄色的"检查说明"区域

### 3. 在WebAdmin验证
1. 进入"检查模板管理"
2. 编辑任意模板
3. 展开检查项，应该能看到"检查项描述"输入框

## 故障排除

如果看不到描述信息：

1. **检查数据库**
   ```bash
   sqlite3 site_manager.db "SELECT COUNT(*) FROM inspection_check_items WHERE description IS NOT NULL AND description != '';"
   ```
   应该显示有描述的检查项数量

2. **重新加载数据**
   - 在App端下拉刷新检查列表
   - 或重新进入检查详情页面

3. **检查模板配置**
   - 在WebAdmin中查看模板
   - 确认检查项有描述内容

4. **重新运行更新脚本**（如果需要）
   ```bash
   cd backend
   python update_existing_check_items_description.py
   ```
