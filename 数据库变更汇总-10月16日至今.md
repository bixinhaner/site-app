## ⚠️ 注意事项

### 1. 枚举值更新
以下枚举需要在应用代码中更新，但不影响数据库结构：

- **InventoryStatusEnum**: 新增 `pending_inspection`, `inspected`
- **WorkOrderStatusEnum**: 新增 `ACTIVATED`
- **TaskTypeEnum**: 新增整个枚举（用于template_bindings）

### 2. 外键约束
`pickup_records` 表的外键从 `task_id` 改为 `work_order_id`。如果有现有数据，需要：
1. 先备份数据
2. 迁移数据（如果task_id和work_order_id有对应关系）
3. 删除旧字段

### 3. JSON字段
SQLite对JSON字段的支持有限，建议：
- 生产环境使用MySQL/PostgreSQL
- 或确保SQLite版本 >= 3.38（支持JSON函数）

### 4. 索引创建
新表的索引在创建表时自动创建，但建议在数据导入后检查索引效果。

### 5. 数据备份
**执行迁移前务必备份数据库！**

```bash
# SQLite备份
cp site_manager.db site_manager.db.backup_$(date +%Y%m%d_%H%M%S)

# MySQL备份
mysqldump -u user -p database > backup_$(date +%Y%m%d_%H%M%S).sql
```

## 🚀 部署步骤

### 方案一：使用迁移脚本（推荐）

1. **备份数据库**
```bash
cp site_manager.db site_manager.db.backup
```

2. **执行迁移脚本**
```bash
# SQLite
sqlite3 site_manager.db < migration_2024_10_16_to_10_22.sql

# MySQL
mysql -u user -p database < migration_2024_10_16_to_10_22.sql
```

3. **验证迁移**
```bash
# 检查新表
sqlite3 site_manager.db ".tables"

# 检查新字段
sqlite3 site_manager.db "PRAGMA table_info(inspection_check_items);"
```

4. **重启应用**
```bash
pkill -f uvicorn
python start_backend.py
```

### 方案二：让SQLAlchemy自动创建（简单但不推荐用于生产）

1. **备份旧数据库**
2. **删除旧数据库** (仅开发环境)
3. **启动应用，SQLAlchemy会自动创建新结构**

```bash
mv site_manager.db site_manager.db.old
python start_backend.py
```

### 方案三：使用Alembic迁移工具（最佳实践）

如果项目已配置Alembic：

```bash
# 生成迁移脚本
alembic revision --autogenerate -m "Add fields and planning tables"

# 查看迁移脚本
cat alembic/versions/xxx_add_fields_and_planning_tables.py

# 执行迁移
alembic upgrade head

# 如需回滚
alembic downgrade -1
```
